# CLAUDE.md — 电磁学科普静态网站

> 本文件为项目级指令,优先级低于全局 `~/.claude/CLAUDE.md`,高于模型默认行为。
> 完整产品设计见 [`design.md`](./design.md),本文件只沉淀**对编码有约束力**的部分。

## 一、项目定位

电磁学科普**纯前端静态网站**:《从琥珀摩擦到 5G 通信:一场跨越三千年的认知革命》。
核心是历史叙事 + 交互可视化 + 内容分层,**无后端、无数据库、可静态托管**(GitHub Pages / Vercel / Netlify)。

三条不可妥协的内容原则(影响实现):
- **真实优先**:完整呈现争议、错误、偶然,不神化。涉及史实需可溯源。
- **分层呈现**:同一知识点须支持「快速 / 深度 / 专业」三层渐进展开。
- **沉浸式**:抽象的"场"与"波"用交互/动画使其可感知。

## 二、技术栈与约束

| 项 | 约定 | 说明 |
|----|------|------|
| 渲染 | 静态站,**无后端** | 所有交互纯前端实现 |
| 框架 | **待确认**(见文末),默认建议 Astro 或原生 HTML/CSS/JS | SEO 友好 + 易静态部署优先 |
| 公式 | KaTeX(优先)或 MathJax | 麦克斯韦方程组等需精确渲染,字体 Latin Modern Math |
| 动画/可视化 | 原生 SVG + Canvas;复杂物理动画可用 D3 / GSAP | 电场线拖拽、电磁波传播、公式拆解 |
| 交互状态 | 闯关进度、动画参数等用 `localStorage` | 无服务端,不持久化到云 |
| 依赖 | 能不引入则不引入,优先轻量 | 拒绝为单一效果引入重型库 |

> ⚠️ 技术栈正式确认前,**不要**擅自 `npm init` 锁定框架。先与用户对齐。

## 三、目录结构(建议,确认框架后细化)

```
/
├── design.md            # 产品设计方案(只读参考,勿改逻辑)
├── CLAUDE.md            # 本文件
├── public/             # 静态资源:历史图片、海报、字体、可下载素材
│   ├── images/
│   └── fonts/
├── src/
│   ├── pages/          # 按板块组织,URL = /板块/文章名
│   ├── components/     # 时间线、信息卡片、公式块等复用组件
│   ├── data/           # 历史事件/科学家/理论的结构化数据(JSON/MD)
│   ├── styles/         # 设计令牌 + 全局样式
│   └── scripts/        # 交互逻辑(电场线、电磁波、闯关)
└── ...
```

内容与表现**强制分离**:史实、人物、理论数据放 `src/data/`,组件只负责渲染。新增内容改数据,不改组件。

## 四、设计系统(编码时直接使用这些令牌,勿用魔法值)

**颜色**

```css
--color-deep-space:   #0F172A; /* 主色 深空蓝 — 理性/宇宙 */
--color-arc-gold:     #F59E0B; /* 主色 电弧金 — 火花/高亮 */
--color-magnet-gray:  #64748B; /* 主色 磁石灰 — 磁的沉稳 */
--color-charge-pos:   #EF4444; /* 正电红 — 正极/正电荷 */
--color-charge-neg:   #3B82F6; /* 负电蓝 — 负极/负电荷 */
```

- **金色 (`--color-arc-gold`) 专用于核心结论/快速阅读层高亮**,不得滥用。
- 正负电荷配色固定红/蓝,强化物理直觉,全站一致。

**字体**

- 标题:Inter Bold;正文:Inter Regular;公式:Latin Modern Math。

**风格**:极简 + 学术感,大量留白,深色背景为主,重点用金色点缀。手绘风历史装置图标(莱顿瓶、伏打电堆、法拉第圆盘、赫兹电火花等)。

## 五、信息架构与路由

七大板块(权重见 `design.md`):首页、交互式时间线(灵魂)、理论的诞生与争议、工程应用史、凡人科学家列传、科学启示录、资源中心。

- **URL 规范**:`/板块/文章名`,语义化、清晰、稳定。
- **内容分层组件**:卡片默认显示第一层(时间/人物/一句话 + 「历史真相」标签),`展开详情` 出第二层,`科学启示` 出第三层。三层结构须可复用。
- **关联跳转**:时间节点 ↔ 理论专题 ↔ 工程案例 ↔ 科学家列传,做成稳定的内部链接关系。

## 六、SEO 与可访问性(硬性要求)

- 每页独立 `<title>` / `meta description` / 关键词。
- **所有图片必须有语义化 `alt`**。
- 语义化 HTML(`<article>` / `<section>` / `<nav>` / 正确标题层级)。
- 横向滚动时间线、动画须提供键盘可达与 `prefers-reduced-motion` 退化方案。
- 模块化设计,预留新增板块(量子电磁学、超导等)与多语言(i18n)扩展位,**不写死中文文案到逻辑里**。

## 七、性能与交互

- 开场动画 3 秒且**可跳过**,不阻塞首屏。
- 长时间线、图片懒加载;素材压缩。
- SVG/Canvas 动画注意帧率与移动端降级。
- 交互(拖动电荷看电场线、调频调幅看电磁波、公式拆解、10 题闯关)全部前端计算,逻辑与渲染分离便于测试。

## 八、科学严谨性(内容代码同等对待)

- 史实标注出处;有争议处并列不同观点,不单方面演绎。
- 物理公式/常量须正确,渲染前核对。
- 「平行宇宙」「如果没有电磁学」等脑洞内容须基于真实历史逻辑,并在 UI 上与史实明确区分,避免误导。

## 九、协作约定

- 应答用中文,术语/变量/代码保留英文。
- 改代码给**完整可替换**内容 + 修改说明;不片段敷衍。
- 不擅自改 `design.md` 的产品决策;如需调整先提示。
- 命名语义化,禁魔法值/重复代码,注释精简。
- **已授权:设计/实现本站时可按需使用任意插件与 skill**(如 prototype 做交互原型、verify/run 跑真实页面、code-review 审查产出等),按任务自主选用。
