# 原型笔记 · 交互式电磁学发展时间线

> ⚠️ 本目录为**一次性原型**,非生产代码。裁决出获胜变体后,按下方「落地」重写进真实项目,然后**删除整个 `prototypes/timeline/`**。

## 要回答的问题

「交互式电磁学发展时间线」(灵魂板块)该长什么结构?5 个时代、时间节点点击弹出**三层信息卡片**(快速 / 详情 / 启示),哪种布局体验最好?

## 三个变体(结构性不同,非换色)

| 键 | 名称 | 主交互 | 信息层级 | 适合 |
|----|------|--------|----------|------|
| **A** | 横向时间轴 | 横向滚动找节点 → 点击**弹浮层**展开三层 | 时间轴为主,卡片为模态 | design.md 原设想;一眼览全程 |
| **B** | 垂直时代分段 | 垂直 `scroll-snap` 逐时代沉浸 → 卡片**就地手风琴**展开 | 时代为一等公民 | 沉浸式叙事;移动端友好 |
| **C** | 双栏 Master-Detail | 左紧凑索引扫描 → 右**常驻详情面板** tab 切三层 | 详情常驻,列表为索引 | 桌面/专业用户快速跳转对比 |

三者共享同一份真实史实数据(`ERAS`,5 时代 16 节点),仅布局/层级/主交互不同。

## 如何查看

零依赖单文件,任选其一:
- 直接双击 `index.html`(`file://` 下变体切换走内存,URL 不更新但功能完整)
- 或 `python -m http.server` 后访问 `http://localhost:8000/prototypes/timeline/?variant=A`(URL 可分享、刷新稳定)

切换:底部浮动栏 `‹ / ›`,或键盘 `←` `→`。变体键 `?variant=A|B|C`。

## 裁决(待用户填写)

- **获胜变体**:__想要 B 的沉浸 + C 的详情常驻___(常见结果是「想要 B 的沉浸 + C 的详情常驻」——若如此请记下混合方案)
- **理由**:__B 的沉浸式叙事, C 的详情常驻__
  - B 的垂直分段,符合移动端友好需求
  - C 的常驻详情面板,符合桌面/专业用户快速跳转对比需求
- **要保留的细节**:__B 的垂直分段, C 的常驻详情面板__

 ## 落地(已定技术栈)

选定后用项目技术栈重写,**不要**直接搬这份原型代码(它在原型约束下写成,无测试、无 a11y 退化):
- **框架**:Astro,时间线做成岛屿组件(交互部分用 React island)
- **样式**:Tailwind CSS,颜色映射到 CLAUDE.md 设计令牌(深空蓝/电弧金/磁石灰/正负电红蓝)
- **滚动**:原生 Scroll Snap(变体 B 已验证可行,与选型一致)
- **数据**:`ERAS` 抽到 `src/data/timeline.*`,组件只渲染(内容/表现分离)
- **a11y**:补 `prefers-reduced-motion`、键盘可达、节点 `aria-*`、语义化标签(原型未做)
- **三层卡片**:抽成可复用组件,供理论/工程/科学家板块共用
